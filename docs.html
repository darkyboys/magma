Documentation of Magma




This Documentation only supports the version of magma you are using.



Introduction to Functions
Functions are very important terms in the Magma's build.magma file as they are responsible for almost everything you will need.

Format for writting a function

You need to follow the following format to write a function:

function name (`value`);

Example:-

main ( `src/main.cc` );



Comments

Comments are ignored by the MAGMA build system but still then needed to be ended with a ;
# This is a comment;


String concatenation

string concatenation is supported which means combining 2 different strings in the function and you can do it like this.
main ( `main` `.cc` ); # this will become main.cc;


Arrays

Arrays are used to give multiple inputs to an Array Function but remember the function must be an Array Function in order to accept arrays. An Array starts with [ and ends with ] brackets.
ArraySyntax [ `Element1` , `Element2` ]; # this is an array
main - Function
main is a very important function as it is used to define your main.cc or main C++ Project file that the magma will compile.

Example :-

main ( `src/main.cc` );

out - Function
out is another very important function as it is used to define your main.cc or main C++ Project file's output binary file name that the magma will create.

Example :-

out ( `main` );

bin - Function
bin is another very important function as it is used to define your binary directory where all the compiled files are going to be stored after compilation. If the directory doesnot exists then it will create it.

Example :-

bin ( `bin` );

compiler - Function
compiler is another very important function as it is used to define your compiler which will be used for compiling your c++ program.

Example :-

compiler ( `g++` );

system - Function
system is a function which is used to define a system call which is nothing but a command which your Operating System will execute. Note that system is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

system ( `ls` );

run - Function
run is a function which is used to define and run a system call live, which is nothing but a command which your Operating System will execute. Note that run is not a sequencer function ,Which are the functions who are directly compiled into IR by the parser. Instead run function runs live without relying on make.

Example :-

run ( `ls` );


Here the main difference between system and run is system runs within the IR, Means within the make and run , Executes without make and IR, Live.

compile_argument - Function
compile_argument is another very important function as it is used to define your arguments which your compiler will use during the compilation process.

Example :-

compile_argument ( `-lcurl` );

clean_main - Function
clean_main is a function which is used to delete your main compiled binary at any point, Also note that clean_main is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

clean_main ();

clean - Function
clean is a function which is used to clear your console at any point, Also note that clean is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

clean ();

exec - Function
exec is a function which is used to execute your main Compiled Binary at any point, Also note that exec is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully. Usually this is used after the main configuration.

Example :-

exec ();

exec_args - Function
exec_args is a function which is used to add the executable arguments to your main Compiled Binary which will be used with the exec function, And be sure to use this before the exec function.

Example :-

exec_args ( `--version` );
exec ();

pkg_config - Function
pkg_config is a function which is used to add the dependency arguments to your main C++ Code which will be used during the compilation of your source code. Here you can define libraries who supports pkg-config.

Example :-

pkg_config ( `--cflags --libs gtk+-3.0` );

combine - Function
combine is a function which is used to combine multiple C++ Source Files with your main C++ Source file which will be done during the compilation of your source code.

Example :-

combine ( `1.cc 2.cc 3.cc` );

make - Function
make is a very very important function which is used to finally build the C++ Binary from the configuration
. Use this function at most last!

Example :-

make ();

include - Array Function
include is a very very important function which is used to set the include path for your compiler which will be the location from where your compiler will look for the headerfiles. This will use the -Ipath to run.

Example :-

include [`path/to/header`];
include [`path/to/header1`, `path/to/header2`];

You can give as many include path as you wish
show_logs - Boolean Function
show_logs is a very usefull function because it takes either `true` or `false` as argument and then either shows or don't shows the build logs , default and undefined value is false. Also remember that true/false must be passed as string.

Example :-

show_logs (`true`);

You can also hide logs anytime for any part you don't need just call the function back with value false.
pkg_in - Array Function
pkg_in is a very very important & advanced function which is used to safely include the dependencies to the program and will only proceed if dependencies are found othervise throw an error (preventing unintended outputs), It uses pkg-config tool but is completely different from pkg_config() function because in pkg_in[], You defines an array of only the names of libraries not flags like --cflags --libs which you would do in pkg_config, here you just defines the name of the libraries directly and it will process all them easily.

Example :-

pkg_in [`gtk+-3.0`, `webkit2gtk-4.0`];
pkg_in [`gstreamer`];

You can give as many libraries as you wish.
increment - Boolean Function
increment is a very very important & advanced function which is used to decrease the build time of your program drastically by allowing you to perform incremental builds , Means magma will only compile your source files if there is any changes in the codebase, Othervise it won't compile your program, This is usefull for time savings but magma always provides option to both opt in and out based on prefrences and it's recommended to use this function for larger projects (Remember larger doesn't means complex, For complex projects plain builds are more recommended rather than incremental for precision).

Example :-

increment (`true`);
increment (`false`); # true by default

If the value is true then it will tell magma to perform incremental builds and note that this checks byte by byte so there is no chance of mistakes in changes detection, Similarly false means you want to tell magma to perform plain and usual builds real time. Remember that this function just opt-in incremental builds, You still have to manually assign the source files manually to compare with snapshots via compare[] array function.
compare - Array Function
compare is a very very important & advanced function which is used to decrease the build time of your program drastically by allowing you to perform incremental builds via increment(`true`);, THis function is used to describe which files to detect or which files you always want to check for building your program. For example you assigned main.cc file to main() function and now you also enables incremental builds with increment() function and you want to only compile main.cc when it has any changes in code then you need to tell magma that hey i want to listen this file changes , And Magma provides you this function to tell itself what to check and what not, Since anything outside of this function won't be checked by the magma.

Example :-

compare [`main.cc`]; # Let's listen to main.cc!
compare [`main.cc`, `Window.cc`, `loading.cc`]; # Thanks to the array function property that we can assign multiple C++ source files directly

You can assign as much files as you wish , Just remember that incremental builds will cancel your entire make(); block if no changes detected for that particular target.
lib - Array Function
lib is a very handy function if you want to include additional libraries which are outside of pkg_in, pkg_config functions, Because this is similar to include but this instead assigns a library flag to your compiler for example i want to make an SDL Application then in that case i have soo many options in magma and one most common option till magma5 was compile_arguments, With that function we could directly add -lSDL for our compiler but now in magma6 we have a dedicated lib array function which can take as many libraries as you wish and will generate flags for them means no more repetative `-lmylibrary1 -lmylibrary2` you can now use magma6 hyper fast speed to automate this for you as now you just say lib [`mylibrary1`, `mylibrary2`]; and you are all set.

Example :-

lib [`SDL2`];
lib [`SDL3`, `gtk3`];

You can give as many libraries as you wish due to properties of magma arrays.
Example build.magma file
main ( `main.cc` );
main_out ( `main` );
bin ( `build` );
compiler ( `g++` );
clean ();
exec ();
make ();
