Documentation of Magma




This Documentation only supports the version of magma you are using.



Introduction to Functions
Functions are very important terms in the Magma's build.magma file as they are responsible for almost everything you will need.

Format for writting a function

You need to follow the following format to write a function:

function name (`value`);

Example:-

main ( `src/main.cc` );



Comments

Comments are ignored by the MAGMA build system but still then needed to be ended with a ;
# This is a comment;


String concatenation

string concatenation is supported which means combining 2 different strings in the function and you can do it like this.
main ( `main` `.cc` ); # this will become main.cc;
main - Function
main is a very important function as it is used to define your main.cc or main C++ Project file that the magma will compile.

Example :-

main ( `src/main.cc` );

out - Function
out is another very important function as it is used to define your main.cc or main C++ Project file's output binary file name that the magma will create.

Example :-

out ( `main` );

bin - Function
bin is another very important function as it is used to define your binary directory where all the compiled files are going to be stored after compilation. If the directory doesnot exists then it will create it.

Example :-

bin ( `bin` );

compiler - Function
compiler is another very important function as it is used to define your compiler which will be used for compiling your c++ program.

Example :-

compiler ( `g++` );

system - Function
system is a function which is used to define a system call which is nothing but a command which your Operating System will execute. Note that system is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

system ( `ls` );

compile_argument - Function
compile_argument is another very important function as it is used to define your arguments which your compiler will use during the compilation process.

Example :-

compile_argument ( `-lcurl` );

clean_main - Function
clean_main is a function which is used to delete your main compiled binary at any point, Also note that clean_main is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

clean_main ();

clean - Function
clean is a function which is used to clear your console at any point, Also note that clean is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully.

Example :-

clean ();

exec - Function
exec is a function which is used to execute your main Compiled Binary at any point, Also note that exec is a sequencer function which are the functions who are directly compiled into IR by the parser. So use is carefully. Usually this is used after the main configuration.

Example :-

exec ();

exec_args - Function
exec_args is a function which is used to add the executable arguments to your main Compiled Binary which will be used with the exec function, And be sure to use this before the exec function.

Example :-

exec_args ( `--version` );
exec ();

pkg_config - Function
pkg_config is a function which is used to add the dependency arguments to your main C++ Code which will be used during the compilation of your source code. Here you can define libraries who supports pkg-config.

Example :-

pkg_config ( `--cflags --libs gtk+-3.0` );

combine - Function
combine is a function which is used to combine multiple C++ Source Files with your main C++ Source file which will be done during the compilation of your source code.

Example :-

combine ( `1.cc 2.cc 3.cc` );

make - Function
make is a very very important function which is used to finally build the C++ Binary from the configuration
. Use this function at most last!

Example :-

make ();

Example build.magma file
main ( `main.cc` );
main_out ( `main` );
bin ( `build` );
compiler ( `g++` );
clean ();
exec ();
make ();
